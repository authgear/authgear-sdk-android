SessionState
Link copied to clipboard

The session state.

An freshly constructed instance has the session state SessionState.UNKNOWN.

After a call to configure, the session state would become SessionState.AUTHENTICATED if a previous session was found, or SessionState.NO_SESSION if such session was not found.

The reason why SessionState is changed.

These reasons can be thought of as the transition of a SessionState, which is described as follows:
